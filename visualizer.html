Algorithm Analysis & Notations

Asymptotic Notations

Big O (O)

Represents the upper bound of the running time. It gives the worst-case complexity.

f(n) = O(g(n)) iff ∃ c, n₀ > 0 s.t. 0 ≤ f(n) ≤ c⋅g(n) ∀ n ≥ n₀

Example: Linear Search is O(n).

Big Omega (Ω)

Represents the lower bound of the running time. It gives the best-case complexity.

f(n) = Ω(g(n)) iff ∃ c, n₀ > 0 s.t. 0 ≤ c⋅g(n) ≤ f(n) ∀ n ≥ n₀

Theta (Θ)

Represents the tight bound. The function grows precisely at the rate of g(n).

f(n) = Θ(g(n)) iff ∃ c₁, c₂, n₀ > 0 s.t. c₁⋅g(n) ≤ f(n) ≤ c₂⋅g(n)

Little o (o)

Represents an upper bound that is not tight. f(n) becomes insignificant relative to g(n) as n → ∞.

Complexity Examples

Space Complexity: Matrix Multiplication

To store three N×N matrices (A, B, and Result), the space required is 3 × N², which is O(N²).

Space Complexity: Factorial

Iterative: Uses a single variable. Complexity: O(1).
Recursive: Uses N stack frames. Complexity: O(N).

Advanced Topics

Fractional Knapsack (Greedy)

Given items with weights and values, maximize total value in a knapsack of capacity W. We can take fractions of items.

Strategy: Calculate Value/Weight ratio for all items. Sort descending. Take as much as possible from the highest ratio item until full.

Time Complexity: O(N log N) (due to sorting).

Minimum Spanning Tree (MST)

Find a subset of edges connecting all vertices in a weighted graph with minimum total weight, without cycles.

  • Prim's Algorithm: Grow MST from a start node. Always add cheapest edge connecting tree to a non-tree node. O(E + V log V).
  • Kruskal's Algorithm: Sort all edges. Add cheapest edge if it doesn't form a cycle (using Union-Find). O(E log E).
